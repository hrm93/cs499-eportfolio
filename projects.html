This project began as a Python-based tool developed in IT 338 to automate spatial analysis workflows in ArcGIS Pro using ArcPy. It was designed for GIS analysts working with pipeline and utility data, enabling quick buffer generation, merging, and export. As the sole developer, I implemented core automation but the original version had limited scalability, required proprietary software, and lacked robust error handling or persistent data storage.

For CS 499, I am reengineering the tool to improve portability, usability, and performance using modern open-source technologies. The enhancements target deeper integration of software engineering principles, algorithmic optimization, and database-driven processing.
